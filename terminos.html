Términos, condiciones y privacidad

Estos términos regulan el uso de MapGestion — el sitio informativo, demos, sandbox y el software con base de datos dedicada que se contrata por cliente. Aquí explicamos qué hacemos, cómo tratamos los datos y qué obligaciones aplican.

1. Aceptación de los términos

Al acceder al sitio, solicitar una demo, usar el sandbox, abrir el mapa real o contratar MapGestion, la persona usuaria o cliente acepta estos términos. Si actúa en nombre de una empresa, declara contar con autorización suficiente para solicitar información, compartir datos operativos y contratar o evaluar el servicio.

Cuando exista un contrato, propuesta, orden de servicio o anexo firmado entre MapGestion y el cliente, ese documento prevalecerá sobre estos términos en lo que resulte expresamente distinto.

2. Alcance del servicio

MapGestion es un software para visualizar, organizar y gestionar mapas, registros, unidades, plazas, estados, usuarios, permisos, bitácoras, incidencias y flujos operativos de un patio o arrendadora.

El servicio se entrega bajo un modelo de instancia dedicada por cliente: cada cliente recibe su propia configuración de software y su propia base de datos independiente. No es un servicio multicliente compartido; los datos de un cliente no se mezclan ni comparten con los de otros clientes.

IncluyeImplementación y mapeo del patio, base de datos dedicada, acceso al software, vistas operativas, roles, registros, historial y soporte según el acuerdo contratado.
No incluye por defectoConsultoría legal, fiscal, laboral, logística, seguros, auditorías regulatorias, hardware, conectividad o captura física de datos en campo.

Las demos, capturas y sandbox pueden usar datos simulados, parciales o de ejemplo. Su finalidad es mostrar el funcionamiento general, no sustituir la validación operativa final de cada cliente.

3. Datos que puede compartir el cliente

Para operar el servicio, solicitar una demo o configurar su instancia, el cliente puede compartir datos de contacto, datos de usuarios, datos operativos y documentación relacionada con su operación.

Datos de contactoNombre, empresa, correo, teléfono, WhatsApp, puesto, plaza o responsable de seguimiento.
Datos de usuariosNombres, correos, roles, permisos, plazas asignadas, historial de acceso y acciones realizadas.
Datos operativosUnidades, placas, identificadores internos, categorías, estados, ubicaciones, mapas, cajones, movimientos, incidencias y bitácoras.
Adjuntos y comentariosNotas, fotografías, archivos, evidencias, audios o documentos que el cliente decida cargar al sistema.
El cliente debe evitar cargar datos personales sensibles, datos financieros, información médica, documentos oficiales o información de terceros que no sea necesaria para el uso operativo de MapGestion, salvo que cuente con autorización y base legal para hacerlo.

4. Privacidad, tratamiento y seguridad de datos

MapGestion tratará los datos compartidos para atender solicitudes, preparar demos, configurar el servicio, operar el software, dar soporte, mantener seguridad, generar historial, cumplir acuerdos comerciales y mejorar el producto.

Cada cliente opera sobre su propia base de datos independiente. Se aplicarán medidas razonables de seguridad administrativa, técnica y organizacional: aislamiento de datos por cliente, control de accesos, permisos por rol, autenticación, bitácoras y prácticas de resguardo acordes al servicio contratado.

La legislación mexicana en materia de datos personales exige un tratamiento legítimo, controlado e informado, y contempla derechos de acceso, rectificación, cancelación y oposición (ARCO). Estos términos se redactan tomando como referencia general la Ley Federal de Protección de Datos Personales en Posesión de los Particulares.

  • El cliente conserva responsabilidad sobre la legalidad, exactitud y autorización de los datos que sube o entrega.
  • MapGestion no vende bases de datos operativas del cliente.
  • MapGestion podrá usar datos agregados, anonimizados o estadísticos para mejorar el producto, siempre que no identifiquen directamente al cliente, usuarios o terceros.
  • Las solicitudes sobre datos personales pueden enviarse a contacto@mapgestion.com

5. Roles, permisos y administración de la instancia

Dentro de su instancia, el cliente administra roles y permisos por usuario, plaza y área. Cada rol define qué puede consultar, editar, aprobar o administrar cada usuario.

  • El cliente designa a uno o más administradores responsables de altas, bajas y asignación de permisos.
  • Los permisos deben otorgarse conforme al principio de mínimo acceso necesario para cada función.
  • El cliente es responsable del uso que sus usuarios hagan de los permisos asignados.
  • MapGestion podrá acceder a la instancia únicamente para soporte, mantenimiento o seguridad, conforme al acuerdo contratado.

6. Obligaciones del cliente y usuarios

  • Proporcionar información verdadera, actualizada y necesaria para la operación del servicio.
  • Contar con permisos, avisos, contratos o autorizaciones necesarias para compartir datos de empleados, colaboradores, clientes, proveedores, unidades o terceros.
  • Administrar correctamente los accesos internos, roles y bajas de usuarios.
  • No compartir contraseñas ni permitir el uso de cuentas por personas no autorizadas.
  • No usar MapGestion para actividades ilícitas, fraudulentas, abusivas o contrarias a derechos de terceros.
  • Validar que los registros, estados, movimientos y reportes reflejen su operación real antes de tomar decisiones críticas.
  • Notificar oportunamente cualquier error, incidente de seguridad, acceso no autorizado o uso indebido detectado.

7. Deslinde de obligaciones y límites de responsabilidad

MapGestion facilita herramientas tecnológicas para organizar información y mejorar trazabilidad, pero no opera físicamente la flota, no administra personal del cliente y no toma decisiones de negocio por el cliente.

En la máxima medida permitida por la ley aplicable, MapGestion no será responsable por:

  • Errores, omisiones o falsedad en los datos capturados por el cliente o sus usuarios.
  • Decisiones operativas, comerciales, legales, fiscales, laborales, de seguros, mantenimiento, renta, venta o logística tomadas por el cliente con base en su propia información.
  • Uso indebido de cuentas, contraseñas, permisos o dispositivos bajo control del cliente.
  • Carga de datos personales o información de terceros sin autorización suficiente.
  • Pérdidas derivadas de fallas de internet, energía, navegador, dispositivos, servicios externos, plataformas de terceros o infraestructura fuera de nuestro control razonable.
  • Daños indirectos, lucro cesante, pérdida de negocio, pérdida de oportunidad, sanciones regulatorias atribuibles al cliente o incumplimientos internos del cliente.
  • Contenido, archivos, fotos, audios, comentarios o evidencia cargada por usuarios del cliente.
MapGestion no sustituye controles internos, auditoría, revisión legal, procesos de seguridad física, verificación humana ni políticas de cumplimiento propias del cliente.

8. Disponibilidad, soporte y cambios técnicos

MapGestion buscará mantener el servicio disponible y funcional, pero no garantiza operación ininterrumpida, libre de errores o libre de fallas externas. Pueden existir ventanas de mantenimiento, actualizaciones, ajustes, mejoras de seguridad o interrupciones por causas fuera de control razonable.

El soporte, tiempos de respuesta, capacitación, respaldos, personalizaciones, integraciones y niveles de servicio dependerán del acuerdo firmado con cada cliente.

9. Propiedad intelectual y uso permitido

El software, interfaz, diseño, código, marca, textos, estructura visual, documentación y materiales de MapGestion pertenecen a sus titulares o licenciantes. El cliente recibe un derecho limitado de uso conforme al servicio contratado, no una cesión de propiedad.

El cliente conserva derechos sobre sus datos operativos, archivos y contenido propio, sin perjuicio de los permisos necesarios para que MapGestion pueda alojarlos, procesarlos, mostrarlos y respaldarlos dentro del servicio.

10. Contratación, licencia y cancelación

Al tratarse de una implementación con instancia y base de datos dedicadas, las condiciones económicas, vigencia, alcance, usuarios incluidos, soporte, implementación, cancelación y renovaciones se definirán en la propuesta comercial, contrato o acuerdo específico con cada cliente (licencia corporativa a la medida).

La falta de pago o incumplimiento material puede causar suspensión del servicio, limitación de accesos o terminación conforme al acuerdo aplicable.
